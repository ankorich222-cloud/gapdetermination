**1. 多个模型管理**

负责模型的加载、切换和生命周期管理。建议用一个工厂模式来支持多模型：
```
ISegmentModel (抽象接口)
├── SAM2Model       // Meta SAM2，适合交互式分割
├── MedSAMModel     // 医学领域微调版SAM
└── CustomModel     // 你们自己训练的模型（ONNX/TensorRT）
```

每个模型内部封装自己的前处理和后处理，对外暴露统一的`Encode` / `Decode`接口。

ONNX怎么暴露encode、decode？用户在过去根据自己标注的得到新的模型，并且报错，第二次加载这个模型呢？

**2. 图像编码缓存层（重要）**

SAM类模型的特点是Image Encoder很重，但Prompt Decoder很轻。用户在一张切片上反复点选时，不应该每次都重新编码图像。
```
CT加载 → 按需对切片做Image Encoding → 缓存embedding（LRU策略）
用户点选 → 直接用缓存的embedding做Prompt Decode → 毫秒级响应
```

这个缓存层是交互体验的关键，建议至少缓存当前切片前后各5-10张的embedding。

**3. 3D传播层**

用户只在少数切片上标注，需要传播到全部切片。几种策略可以组合：

- **帧间传播**：以有prompt的切片为锚点，向相邻切片传播（SAM2本身支持视频模式，但是我们应用场景为CT切片）
- **连通域约束**：用3D连通域分析过滤掉游离的噪声区域
- **切片间形变**：相邻CT切片器官变化平滑，可以用简单的形变场做引导
